微前端核心：JS 沙箱 / CSS 隔离（真实可运行）

多个子应用共享同一个 window / document，全局变量与 CSS 会互相污染。下面三个例子分别演示：Proxy 代理沙箱、快照沙箱、Shadow DOM 样式隔离。 点按钮，结果会打印到黑框里。

① Proxy 代理沙箱 ProxySandbox

用 Proxy 拦截 window 的读写：写操作落在 fakeWindow 上， 不污染真实 window；支持多实例（每个子应用一个独立 fakeWindow）。

② 快照沙箱 SnapshotSandbox

激活时给整个 window 拍一张「快照」，子应用运行期间可随意改真实 window； 卸载时对比快照，把被改动/新增的属性还原。兼容不支持 Proxy 的老环境， 缺点：直接操作真实 window，同一时刻只能有一个沙箱激活。

③ Shadow DOM 样式隔离（strictStyleIsolation）

外部有一个 .box（灰色虚线框）。我们用 attachShadow 创建一个 Shadow DOM，内部也放一个 .box 并写完全不同的样式。 由于 Shadow DOM 天然隔离，内部样式不会外泄、外部样式也进不去。
