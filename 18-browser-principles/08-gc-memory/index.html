🗑️ GC 与内存泄漏演示台

点击「制造泄漏」会不断分配 脱离文档的 DOM 节点（被全局数组持有）+ 未清理的定时器闭包（持有大数组）。这些对象「没用了却仍从 GC root 可达」，GC 无法回收 —— 这就是泄漏。用 F12 → Memory 面板抓快照对比即可看到。

🔬 DevTools 排查步骤

  1. F12 → Memory 面板 → Heap snapshot → 抓第 1 张（基线）。
  2. 多次点「💥 制造泄漏」，让上面的计数涨上去。
  3. Memory 面板点垃圾桶手动 GC → 抓第 2 张 → 顶部切 Comparison 与第 1 张对比。
  4. 看 Delta 列：Detached HTMLDivElement、(array) 只增不减 = 泄漏证据；过滤框输入 Detached 可看引用链 Retainers。
  5. 点「✅ 正确释放」→ 再手动 GC → 抓第 3 张，Detached 应归零，泄漏被解除。
