微前端 · 依赖共享 / 版本管理

用浏览器原生 import maps 让多个子应用共享同一份依赖实例（单例），并对比“各自加载副本”的区别。

🗺️ 本页用的 import map

<script type="importmap">
{
  "imports": {
    "shared-lib": "./shared.js"   // 裸名 shared-lib 统一映射到这一个文件
  }
}
</script>

子应用 A、子应用 B、主入口都 import ... from 'shared-lib'，因此拿到同一个 URL、同一份单例。
子应用 C 故意不走 import map，直接 import './shared-standalone.js'（另一个文件）——它是独立副本，用来做反面对比。

📜 运行结果（看 instanceId 是否相同、count 是否互通）

预期：A / B / main 的 instanceId 完全相同，共享 count 累加到 3（A 加 2 次、B 加 1 次，B 还能读到 A 加过的值）； 而 C 的 instanceId 不同、count 从 0 独立开始 —— 证明 C 多加载了一份重复依赖，状态互不相通。 打开浏览器控制台还能看到 [shared.js] 模块被执行 只打印一次（同一 URL 只执行一次）， [shared-standalone.js] 单独打印一次。
