Web Worker · 主线程卸载对比 Demo

同一个 CPU 密集计算（统计 300 万以内素数），对比「主线程直接算」与「Worker 计算」时主线程的流畅度差异。

👀 观察方法：盯着左侧旋转方块、计数器（每 100ms +1）和 FPS。
点「主线程直接算」→ 方块僵住、计数器停更、FPS 掉到个位数（页面卡死几秒）。
点「Web Worker 计算」→ 方块照转、计数器照涨、FPS 保持约 60（主线程毫无阻塞）。

主线程流畅度探针

0
计数器（每 100ms +1）
–
FPS（每秒帧数）

A. 主线程直接算 ❌

把 CPU 密集计算放在主线程，形成长任务（Long Task），独占主线程数秒，UI 全冻结。

B. Web Worker 计算 ✅

把计算 postMessage 给 worker 线程执行，主线程只负责收发消息，UI 始终流畅。
