爬虫 / 社交抓取器到底看到了什么？

下面是两种应用「服务器直出的原始 HTML 源码」对照。抓取器通常不执行 JS，它拿到的就是这份源码本身。

❌ CSR 应用返回的 HTML（空壳）

<!-- 服务器直出：正文根本不在源码里 -->
<!doctype html>
<html>
  <head>
    <title>My App</title>  <!-- 通用标题，非本页专属 -->
    <!-- 没有 description、没有 og:image -->
  </head>
  <body>
    <div id="root"></div>  ← 空的！
    <script src="/bundle.js"></script>
    <!-- 正文/meta 要等这段 JS 下载+执行+取数
         之后才被塞进 #root。抓取器不跑 JS = 看不到 -->
  </body>
</html>
抓取器解析结果：标题是通用的「My App」、无描述、无 OG 图、无正文。 SEO 收录残缺，分享卡片没图没标题。

✅ SSR / SSG 应用返回的 HTML（完整）

<!-- 服务器直出：正文与 meta 已全部在源码里 -->
<!doctype html>
<html>
  <head>
    <title>iPhone 20 Pro 开箱评测</title>
    <meta name="description" content="全网首发详评…">
    <meta property="og:title" content="开箱评测">
    <meta property="og:image" content="…/og.png">
  </head>
  <body>
    <article>
      <h1>iPhone 20 Pro 开箱评测</h1>
      <p>今天我们拿到了…</p>  ← 正文已在！
    </article>
    <script src="/bundle.js"></script>
    <!-- JS 仅用于「水合」让页面可交互，
         内容早已随 HTML 直出 -->
  </body>
</html>
抓取器解析结果：本页专属标题、描述、OG 大图、完整正文一次拿全。 SEO 收录完整，分享卡片有图有标题。无需执行任何 JS。

为什么 CSR 站的分享卡片常常「只有一个链接，没图没标题」？

当你把链接粘到微信 / 微博 / Slack / Discord 时，平台的抓取器会请求你的页面并读取 og:title / og:image 来生成卡片。这些抓取器 几乎都不执行 JavaScript。

CSR 站的 OG 标签是 JS 运行后才插入 <head> 的——抓取器看不到，于是卡片空白。 SSR/SSG 站的 OG 标签随 HTML 直出，抓取器一读就有，卡片完整。

动手验证：对任意网页「右键 → 查看网页源代码」，在源码里搜 og:image 和正文关键字。 搜得到 = 服务端直出（SEO 友好）；搜不到但页面上有 = JS 后插入（抓取器可能看不到）。
