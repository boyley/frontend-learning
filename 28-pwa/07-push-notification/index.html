🔔 推送通知（Push Notification）

权限 → 订阅 → 推送 → 展示 → 点击

① 请求通知权限（Notifications API）

权限状态：default

浏览器只在用户手势触发时才允许弹权限框；拒绝后无法再用代码重新弹出。

② 本地通知（无需服务器）

直接用 registration.showNotification() 弹一条通知，验证展示与点击逻辑。

③ 订阅 Web Push（PushManager）

用 VAPID 公钥向浏览器推送服务订阅，得到一个 PushSubscription（含 endpoint）。真实推送需把它发给你的服务器保存。

（尚未订阅）

④ 测试 push 事件

没有真实推送服务器时，在 DevTools → Application → Service Workers 的「Push」框里填入任意文本点 Push，即可派发一次 push 事件，触发 sw.js 里的处理并弹通知。
