♻️ Service Worker 生命周期

install → (waiting) → activate → controlling → fetch

Service Worker 有一套严格的生命周期，用来保证「同一时刻只有一个版本控制页面」。下面的状态条会随事件点亮，右侧日志由 sw.js 通过 postMessage 实时推送。

① install预缓存资源
② waiting等待旧版退出
③ activate清理旧缓存
④ controlling接管页面
⑤ redundant被新版替换

体验「更新流程」：把 sw.js 里的 VERSION = 'v1' 改成 'v2' 保存，回到本页点「检查更新」→ 新 SW 进入 waiting（旧版仍在控制）→ 点「跳过等待」让它立即 activate。
