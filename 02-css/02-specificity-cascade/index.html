优先级、层叠与 !important

层叠（Cascade）决定多条规则冲突时谁生效，顺序为：来源/重要性 → 优先级 → 源码顺序。

演示一：同一元素被多条规则命中

我的最终颜色由“优先级最高”的规则决定（当前为橙色 = 行内样式）

规则选择器权重 (行内,id,类,元素)颜色
Ap(0,0,0,1) 灰
B.note(0,0,1,0) 绿
C#msg(0,1,0,0) 蓝
行内style=""(1,0,0,0) 橙 ← 胜出

逐位比较：行内的 1 大于 id 的 0，直接胜出，后面位不用再比。

演示二：!important 强行翻盘

即使写了行内橙色，!important 仍让我变成红色（这就是它危险的原因）

普通比较里行内最高，但带 !important 的声明属于更高的“重要性层级”，会越过普通声明。
